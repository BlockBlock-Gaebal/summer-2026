## 결정 순서 (이 순서대로 진행하면 30~40분 컷)

### 1️⃣ [최우선] 안건 ⑤ — "성공"의 정의
- 30일 중 29일 성공한 사람은 성공인가 실패인가?
- 옵션: (a) 임계값 방식 (예: 85% 이상 = 성공, 챌린저스 방식) / (b) grace day (봐주는 날 n일) / (c) 실패 1회당 부분 몰수
- **왜 최우선**: 이게 정해져야 커브 입력값 "탈락일 d"의 의미가 확정됨. 뒤 안건 전부의 선행 조건.

### 2️⃣ 안건 ⑩ — 커브 인덱싱 (⑤랑 묶어서, 같은 "d의 정의" 문제)
- 현재: day D(마지막 날) 탈락해도 전액 환급 → **마지막 날 체크인의 금전 가치가 0** (무임승차 구멍)
- 제안: 커브 입력을 "탈락일 d" → **"완주일수 d−1"**로 변경
- 비용: 테스트 기대값 표(T1/T2/T3/V4) 전부 재계산 필요 — 영향 범위 최대 안건이라 지금 확정해야 함

### 3️⃣ 안건 ③ — 커브 α 값
- 커브: `r(d) = α·(d/D) + (1−α)·(d/D)²` (α = 볼록한 정도, 유일한 튜닝 파라미터)
- **제안: α = 0.3** (탐색 구간 [0.2, 0.4], 시뮬레이션 근거는 설계문서 §9-6)
- 나중에 바꿔도 됨(코드상 파라미터) — 발표용 기준값만 정하면 됨

### 4️⃣ 안건 ① + ④ — 엣지 케이스 2종
- ① 전원 탈락(챌린지 불성립) → **원금 전액 반환** (방향 확정됨, 승인만)
- ④ 전멸(최후 생존자까지 동시 탈락) → 현재 임시 규칙: 조기 종료 + 커브 환급 확정 + 미방출 스트림은 vault 잔류. 이대로 갈지 "수령자 없는 몰수는 불성립" 안으로 갈지

### 5️⃣ 안건 ② + ⑪ — 승인성 안건 (각 30초)
- ② **스트리밍 분배** 채택 — 몰수금을 즉시 배당 대신 잔여 기간에 균등 방출. 배당이 단조증가하게 되어 후반 이탈 유인 차단 (강력 권고, 설계문서 명제로 증명됨)
- ⑪ **배당 절단 규약** — "정산 시 1회 절단"으로 명문화 (dust 감소, 유저에게 유리. 구현 이미 이 방식)

### 6️⃣ 안건 ⑥ — 보고 사항 (결정 아님)
- 중도참여자 커브는 **개인 잔여기간으로 정규화** — 안 하면 완주해도 손해 보는 "기부 함정" 발생 (수학적으로 강제됨, 설계문서 명제 참고)

### 7️⃣ [여유 있으면] 안건 ⑦⑧⑨ — ver4/로드맵
- ⑦ MVP 중도참여 = 개방형 + 참여 마감선. **τ(마감선) 제안: 기간 50%**
- ⑧ 공개방 로드맵 우선순위: **빈티지 분리** > NAV 진입가 (제안)
- ⑨ ver4를 최종 MVP에 포함할지 — ver3 진행 속도 보고 판단 (구현은 이미 됐으니 "테스트넷까지 끌고 갈지"의 문제)
