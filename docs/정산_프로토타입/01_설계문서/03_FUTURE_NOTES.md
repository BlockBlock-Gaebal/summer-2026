## 2. 커브 lookup table 업그레이드 경로 (ver5 config 후보)

- **뭐**: 커브를 2차 다항식 고정이 아니라 **구간별 선형 lookup table**(u64 브레이크포인트 vector)로 일반화
- **왜**: 나중에 실측 hazard에서 역산한 임의 형태의 커브를 코드 수정 없이 꽂을 수 있음. 온체인 비용도 저렴 (구간 탐색 + 선형 보간)
- **어떻게**: ver5 config에 `curve_mode: poly(alpha_bp) | table(breakpoints)` 분기 하나. calc_refund가 순수 함수로 격리돼 있으므로 함수 교체 하나로 끝 — 현 설계가 이미 이 확장을 준비해둔 구조
- **전제**: 유저 커스터마이징은 여전히 "수식 직접 입력"이 아니라 "미리 검증된 테이블 선택" (PROJECT_CONTEXT §11-3 원칙 유지)

## 3. 탈락자 흑자 cap 토글 (ver5 config 후보, 디폴트 OFF)

- **배경**: 초반 대량 탈락 방에서 후반 탈락자는 환급+누적배당 > 원금 가능. 버그 아님 — 시간가중의 정의 그 자체 (25일 생존의 대가). 완주자는 항상 그 이상 받으므로 (MON) 유지. **팀 입장: 흑자 허용이 기본, 원하는 방은 끌 수 있게**
- **구현 형태는 하나로 강제됨** — "이미 적립된 배당은 회수하지 않는다"(톤틴 비가역성)가 설계 기둥이므로 **배당 클로백 금지**. 따라서 환급 쪽만 깎는다:
  ```
  cap ON일 때 탈락 시점:
  환급_capped = min( calc_refund(...), max(0, stake − 누적배당) )
  깎인 차액은 그날 몰수액에 합산 → 스트림으로 방출
  ```
- **성질**: (a) 배당 클로백 없음 (b) 탈락자 총수령 ≤ stake (c) 차액이 스트림으로 가므로 보존 법칙 유지 (d) 탈락일별 총수령 단조성 유지 (e) 보너스: cap 근처 참가자에게 "탈락하면 초과분 날림" → 계속 유인 강화
- **주의 ①**: 누적배당만으로 이미 stake 초과한 극단 케이스는 환급 0으로 깎아도 흑자 잔존 — 클로백 금지의 대가로 수용 (발생 확률 낮음)
- **주의 ②**: 환급이 경로 의존이 됨 (calc_refund에 누적배당 입력 추가 → 순수성 약화, 테스트 기대값이 시나리오 의존). 프로토 복잡도 상승의 주범이라 디폴트 OFF + ver5로 미루는 것
- config 이름 후보: `cap_dropout_payout: bool`

## 4. UI 리프레임: "탈락 시 환급 커브" → "일일 베스팅" (컨트랙트 무변경)

- r(d)가 증가함수라는 것은 **매일 스테이크가 조금씩 베스팅되어 돌아온다**와 수학적으로 동치
- 전망이론의 손실 구간 체감성(diminishing sensitivity) → 작은 손실/회수 여러 번이 큰 한 방보다 심리적 체감 큼 (loss segregation)
- FE에서 "오늘 버티면 ○○원 회수 확정 + drip 수령"으로 표시 — β-δ 유저에게 체감 강도 최대화. **수식·컨트랙트는 1비트도 안 바뀜, 순수 UI 프레임**
- Phase 2 FE 설계 시 반영

## 5. α 캘리브레이션 (PROTO_SPEC §8과 연동 — 진모 몫)

- "더 좋은 커브"의 실체 = 새 함수가 아니라 **α를 실측 h(d)로 캘리브레이션**하는 작업
- 이론적 목표: 생존 임금 w(d)가 노력 비용 c(d)를 "딱 필요한 만큼만" 이기도록 (과잉 볼록의 후생 비용 최소화)
- 주의: Lally 2010 — 습관 자동화(중앙값 66일) 형성 후 c(d)가 꺾일 수 있음 → 장기 챌린지(60일+)에선 w를 계속 올리는 게 과잉일 수 있음. D별로 α를 달리하는 것도 옵션 (D 짧으면 α↑, 길면 재검토)

## 6. 발표용 추가 인용각 (킬러 레퍼런스 보강)

- **Becker & Stigler (1974)** "Law Enforcement, Malfeasance, and Compensation of Enforcers" — 성과 본드(performance bond): 이탈/배신 시 몰수되는 예치금 + 근속 시 이연 보상. **우리 커브의 정확한 학술적 족보** (Lazear 1979 이연 임금이 그 연장선 — CURVE_DESIGN §8에 이미 있음)
- **Kivetz, Urminsky & Zheng (2006)** "The Goal-Gradient Hypothesis Resurrected", JMR — 목표에 가까울수록 노력 가속(커피 스탬프카드 실증). **볼록 커브의 행동 실증 근거**
- 최적계약이론 프레임: 현재편향(β-δ) + 일일 유혹 충격 하에서 최적 계약의 임금 스케줄은 증가함수 → "증가하는 w(d)는 우리가 고른 게 아니라 이론이 도출하는 형태, 2차식은 그 최소 파라미터 구현"

## ⚠️ 별도 안건 (TMI 아님 — 프로토 구현 전 확정 필요, 회의행)

**마지막 날 무임승차 (커브 인덱싱 off-by-one)**: 현 정의 r(d/D)는 day D 탈락 시 r(1)=1 → 전액 환급, 마지막 날 체크인의 금전 가치가 0에 수렴 (전원 성실 방이면 정확히 0). 처방: 커브 입력을 "탈락일 d" → **"완주일수 d−1"**로 (`환급 = stake × r((d−1)/D)`). day 1 탈락=r(0)≈0, day D 탈락=r((D−1)/D)<1, **완주만 r(1)=1 도달**. 명제 1~4 전부 유지. 대가: T1/T2/T3 기대값 표 전면 재계산 필요 → "성공의 정의" 안건(탈락일 d의 정의)과 묶어서 회의 확정 후 시뮬레이터로 표 재생성. ver4 적용 시 개인 타임라인(D_j) 기준 완주일수로 동일하게.
