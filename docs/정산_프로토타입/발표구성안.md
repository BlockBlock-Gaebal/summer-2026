## 슬라이드별 상세

### S1. 훅 — "습관 앱이 돈을 쥐면 생기는 일"
- GymPact(미국): 실패자 벌금 → 성공자 분배 앱. 회사가 예치금 쥐고 판정하다 **성공자에게도 과금** → FTC 제재(약 $95만 소비자 환불 명령) → 폐업
- 멘트: "커밋먼트 앱의 아이디어는 검증됐는데, **돈과 판정을 중앙이 쥐는 구조**가 무너졌습니다"
- 한국 챌린저스: 같은 구조로 누적 거래액 3000억+ — 시장 수요는 증명됨. 신뢰 구조만 문제

### S2. 우리 답 — 한 줄
- "저희는 그 **커스터디와 정산 규칙을 스마트 컨트랙트에 박제**합니다. 운영자인 저희도 못 건드립니다"
- 그림: 참여자 → Vault(컨트랙트) → 체크인 → 몰수/분배 플로우 (기획안 2-1 다이어그램 재활용)
- 이론 한 줄: 커밋먼트 디바이스 × 손실 회피(잃는 고통 ≈ 얻는 기쁨의 2배)

### S3. 경쟁 지형 — 그래서 뭐가 새로운데?
- 표 축약판: 챌린저스(중앙화) / GymPact(폐업) / ETHGlobal 습관 앱들(전부 flat 몰수)
- **펀치라인**: "온체인 습관 앱 선례를 전수 조사했는데, **탈락 '시점'을 정산에 반영한 프로젝트는 없었습니다.** day1 탈락과 day29 탈락을 똑같이 취급합니다. 여기가 저희의 white space"

### S4. 핵심 메커니즘 — 두 개의 레이어
- ① **환급 커브**: `r(d) = α·(d/D) + (1−α)·(d/D)²` — 오래 버틸수록 많이 돌려받음
  - 그래프: `01_retention_curves.png` + `02_staying_wage.png`
  - 멘트 포인트: "커브 방향을 감으로 고른 게 아니라, **'하루 더 버티는 것의 가치(생존 임금)가 노력 비용을 이겨야 한다'는 인센티브 조건을 정의하고 그걸 만족하는 함수족을 유도**했습니다" ← 이 문장이 팀의 무기
- ② **스트리밍 분배**: 몰수금을 즉시 배당 X, 잔여 기간에 균등 방출
  - 그래프: `03_flow_compare.png`
  - 멘트: "즉시 배당은 초반 대박·후반 쭉정이 구조라 후반 이탈을 유인합니다. 스트리밍은 배당이 **수학적으로 단조증가** — 버틸수록 시급이 오릅니다"
- (여유 있으면) Q&A 대비 인용각: Becker-Stigler(1974) 효율임금, Kivetz(2006) 목표 가속

### S5. ★ 킬러 슬라이드 — 표 하나로 끝내기 (T1 시나리오)
- 설정: A, B, C 각 10 SUI, 5일 챌린지. B가 day2, C가 day4 탈락, A 완주

| | 예치 | 탈락 | 커브 환급 | 받은 배당 | 최종 수령 |
|---|---|---|---|---|---|
| A | 10 | 완주 | — | +6.5 | **16.5** |
| C | 10 | day4 | 8.0 | +1.5 | **9.5** |
| B | 10 | day2 | 4.0 | 0 | **4.0** |
| 합 | 30 | | | | **30 (보존 ✓)** |

- 여기서 짚을 것 3개 (이 표가 설계 전부를 담고 있음):
  1. **B ≠ C**: 같은 "탈락"인데 수령액이 다름 = 시간가중 (기존 앱은 전부 둘 다 0원)
  2. **C의 배당 1.5는 탈락해도 유지**: "버틴 기간의 기여는 인정" — 일일정산의 정의
  3. **합계 30 = 30**: 돈은 새지도 생기지도 않음 — 이게 컨트랙트 신뢰의 실체
- 멘트: "이 표의 숫자 하나하나가 온체인 테스트로 검증된 값입니다" → S6으로 연결

### S6. 라이브 데모 — 테스트 실행이 곧 데모
- 터미널 띄우고: `cd contracts/godsaeng && sui move test`
- **21/21 초록불**이 뜨는 화면 자체가 데모. 멘트: "방 생성→3명 참여→매일 정산→claim→잔액 검증까지 풀시나리오가 이 안에 있습니다"
- 백업 플랜(발표장 네트워크/노트북 리스크): 실행 화면 **미리 녹화 or 스크린샷** 준비. 라이브 실패 시 즉시 전환
- 보조 화면: 커밋 히스토리 (`git log --oneline`) — ver1→ver4 진화가 커밋 5개로 보임 = "점진 개발 과정" 증명

### S7. 로드맵 + 마무리
- 구현됨(✅): 정산 코어 ver1~4 / 다음(4주 내): Testnet 배포 + FE 지갑연동 + BE 오라클 / 로드맵(발표만): 인증 강화(사진+상호확인→AI), zkLogin, 빈티지 분리 공개방, No-Loss 모드
- 마무리 멘트: "돈 로직은 이미 돌아갑니다. 남은 4주는 이걸 사람이 쓸 수 있게 만드는 시간입니다"

## Q&A 예상 문답 (기존 논리 + 신규)

- **"챌린저스랑 뭐가 다르냐"** → 챌린저스는 회사가 예치금·판정·수수료를 통제. GymPact가 그 구조로 폐업. 우리는 커스터디+정산을 컨트랙트가 집행 + 챌린저스가 못 하는 시간가중 차등 몰수
- **"도박 아니냐"** → 도박은 우연에 걺. 이건 본인이 100% 통제하는 노력에 거는 자기구속 계약
- **"인증은 어떻게 하냐"** → "돈은 온체인, 인증은 오프체인" 하이브리드. 오라클 인터페이스만 열어둬서 인증 방식이 뭐로 진화하든 컨트랙트 불변. 판정 중앙화는 인정하는 타협 (Proof of Workout도 Strava 의존) — 로드맵에 다수 서명/낙관적 이의제기
- **"왜 Sui냐"** → 돈이 mapping 속 숫자가 아니라 리소스 객체(Coin) — 복사·소실이 언어 차원에서 불가. Vault 예치→분할→분배가 핵심인 프로젝트와 궁합. + zkLogin으로 UX 로드맵
- **"마지막 날에 그냥 안 하면?"** → (안건 ⑩ 확정 전이면) "정확한 지적. 인덱싱을 완주일수 기준으로 수정하는 안건이 이미 등재돼 있음" — 오히려 설계 검증이 살아있다는 어필로 전환 가능
