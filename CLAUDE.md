# CLAUDE.md — 갓생 내기 DeFi 프로토콜 (팀 공통 컨텍스트)

> 이 파일은 저장소 루트에 위치하며, 팀원 누구든 Claude Code를 실행하면 자동으로 로드된다.
> **상세 배경/설계 근거는 `docs/PROJECT_CONTEXT.md`를 읽을 것** (리서치 결과, 경쟁 분석, 설계 논쟁 전체 포함).

## 1. 프로젝트 한 줄 요약

**의지력에 돈을 거는 탈중앙 습관 챌린지 프로토콜.** 참여자들이 챌린지(예: 미라클 모닝) 방에 돈을 예치하고, 실패하면 예치금이 몰수되어 성공자들에게 분배된다. 핵심 차별점: **시간가중 차등 몰수(day1 탈락 ≠ day50 탈락) 등 복잡한 정산 수식을 스마트 컨트랙트로 신뢰 불필요하게 자동 집행** — 이게 없으면 블록체인을 쓸 이유가 없다.

## 2. 팀 & 목표

- 연세대 블록체인 동아리 블록블록, 3인 팀 (하진모, 김유안, 정승준)
- 전원 개발 첫 입문 (크립토 도메인 지식은 헤비유저 수준), AI 협업(바이브 코딩) 중심
- 목표: 여름방학 4주 내 **첫 해커톤 MVP**. 토큰 발행/수익화 목적 없음
- 최대 리스크는 기술 부족이 아니라 **스코프 폭발**. 모든 설계 판단 기준 = "발표 5분 안에 데모 가능한가"

## 3. 기술 스택 (확정)

| 레이어 | 스택 |
|---|---|
| 체인 | **Sui** (Devnet/Testnet, faucet으로 테스트 SUI) |
| 컨트랙트 | **Sui Move** (2024 에디션), Sui CLI (`sui move build/test`, `sui client publish`) |
| FE | React + TypeScript (Vite), `@mysten/dapp-kit` + `@mysten/sui`로 지갑 연동 |
| BE | NestJS (TypeScript) |
| 환경 | 전원 WSL(Ubuntu), Node v20, yarn, VS Code + WSL 확장 |
| 협업 | GitHub `BlockBlock-Gaebal/summer-2026`, 본인 이름 브랜치 → PR → main |

## 4. 아키텍처 원칙 (확정)

1. **돈은 온체인, 인증은 오프체인**: 예치/정산수식/claim은 Move 컨트랙트. 체크인 인증 판정은 BE가 오라클로 결과 제출. 인증 상세 설계는 후순위 — **오라클 주소 인터페이스만 열어두면 나중에 뭘 붙여도(BE 자동화/커뮤니티투표/AI) 컨트랙트 불변**
2. **커스텀 챌린지 방**: 유저가 방을 팜 (`create_challenge(체크인시간, 윈도우, 기간, 예치금, 공개여부)`). Challenge = shared object 하나. 챌린지 타입은 "시간 기반 체크인"으로 한정 (헬스 등 검증 불가 유형 제외)
3. **정산 수식은 무조건 온체인** + 순수 함수로 분리 (테스트 = 수식 검산). Move는 소수점 없음 → u64 스케일링, 복잡한 커브는 구간별 선형 근사
4. **pull 패턴 정산**: 일괄 분배 루프 금지, 각자 `claim` 호출
5. **데모 모드**: 기간/윈도우는 생성 파라미터 (하드코딩 금지) → "1 day = 30초" 챌린지 가능해야 함
6. 시간은 `Clock` 객체 (UTC 밀리초, 한국 6시 = UTC 21시 — KST 보정 주의)

## 5. Claude(AI)를 위한 작업 지침

- 팀원은 전원 개발 입문자: 코드에 개념 설명을 곁들이고, 단계를 뭉개지 마라. 단 DeFi 도메인(AMM/스테이킹/슬래싱/LP지분)은 설명 없이 써도 됨
- **언어는 Sui Move다 (Solidity 아님).** Move 패턴(객체 소유권, Coin 리소스, shared object, Clock)은 처음 나올 때 "왜"를 짚어라
- Sui Move는 학습 데이터가 적어 옛 문법이 섞일 수 있다 → 2024 에디션 기준으로 작성, 불확실하면 docs.sui.io 확인 권고
- 스코프 확장 제안은 신중하게. 로드맵 밖 기능을 먼저 제안하지 마라
- 세션 종료 시 요청받으면 아래 "현재 상태" 섹션을 갱신하라

## 6. 팀 운영 룰

1. **주 1회 회의** → 결정사항은 이 파일 + docs에 즉시 반영 (PR로. 갱신 담당: 진모)
2. **30분 룰**: 혼자 30분 막히면 단톡방에 던진다
3. **PR은 작게**: 마일스톤 하나 = PR 하나, 리뷰 1명 이상, main은 항상 테스트 통과 상태
4. **AI 코드 설명 룰**: AI가 짠 코드는 머지 전 본인이 팀원에게 말로 설명 가능해야 함. 못 하면 AI에게 한 줄씩 해설시켜 이해 후 머지
5. 개인 컨텍스트는 `CLAUDE.local.md` (.gitignore 등록) — 담당 파트, 개인 선호 등

## 7. 개발 로드맵 (Phase 1: 돈 로직 우선)

| 마일스톤 | 내용 | 완료 기준 |
|---|---|---|
| **M0** | Sui CLI 설치, `contracts/` 패키지 초기화 | 전원 로컬에서 `sui move test` 초록불 |
| **M1** | `create_challenge` + `join` (Coin 예치) | "3명 참여 시 vault = 예치금×3" 테스트 통과 |
| **M2** | `submit_results` (오라클 전용, Clock 검증) | 권한/중복제출 테스트 통과 |
| **M3** | 정산 수식 — 볼록결합 커브 + 스트리밍 (설계 완료, α만 회의 확정 대기) | 시뮬레이터 기대값 표 대조 + **보존 법칙(∑수령+dust=∑예치)** 통과 |
| **M4** | `claim` (pull 패턴) | 이중 claim 방지 테스트 통과 |
| **M5** | 데모 모드 검증 ("1 day = 30초") | 풀시나리오 테스트(생성→참여→시뮬→정산→잔액검증) 통과 |

- 주차 계획: 1주차 M0~M1(몹 프로그래밍) / 2주차 M2~M3 / 3주차 M4~M5 + Testnet 배포 → FE/BE 착수 / 4주차 통합+데모
- 협업 방식: M1은 셋이 몹 프로그래밍(컨벤션 확정), M2부터 "구현/테스트작성/리뷰" 롤을 마일스톤마다 교차
- 학습: 크립토좀비(Solidity) 대신 **docs.sui.io 튜토리얼**
